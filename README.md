# Introduction to Logs

## Objective

Learn what logs are and why they are important in Cybersecurity and Security Operations Centers (SOC).

---

## What is a Log?

A log is a record of an event that occurs on a computer system, application, or network.

SOC Analysts use logs to investigate suspicious activities and identify security incidents.

---

## Example Log

Time: 14:30

User: john

Action: Login

Status: Failed

IP Address: 10.0.0.25

---

## Information Found in a Log

- Time
- Username
- Action Performed
- Status (Success or Failed)
- Source IP Address

---

## Why Logs are Important

Logs help security analysts:

- Detect brute-force attacks
- Investigate suspicious logins
- Monitor user activities
- Identify malware infections
- Perform incident investigations

---

## Example Security Scenario

If a user fails to log in over 100 times within five minutes from the same IP address, it may indicate a brute-force attack.

---

## Key Takeaways

- Logs record events.
- Every event has useful information.
- SOC Analysts investigate logs every day.
- SIEM tools collect and analyze logs automatically.

---

## Skills Learned

- Understanding Logs
- Reading Login Events
- Identifying Failed Logins
- Recognizing Brute-force Attacks

Windows Event Logs

## Objective

Learn the basics of Windows Event Logs and their importance in Security Operations Centers (SOC).

---

## What are Windows Event Logs?

Windows Event Logs record activities that occur on a Windows operating system.

These logs help administrators and SOC Analysts monitor systems, troubleshoot problems, and investigate security incidents.

---

## The Five Main Windows Logs

### 1. Application

Records events generated by applications.

Examples:
- Application crashes
- Installation errors
- Software warnings

### 2. Security

Records security-related events.

Examples:
- Successful logins
- Failed logins
- Password changes
- User account creation

### 3. System

Records events generated by Windows.

Examples:
- Driver issues
- Hardware failures
- Windows startup
- Windows shutdown

### 4. Setup

Records Windows installation and update events.

### 5. Forwarded Events

Stores logs collected from remote computers.

---

## Important Event IDs

4624 - Successful Login

4625 - Failed Login

4634 - User Logged Off

4720 - User Account Created

4726 - User Account Deleted

---

## Why SOC Analysts Use Event Logs

- Detect brute-force attacks
- Investigate login attempts
- Monitor system health
- Track user activity
- Investigate security incidents

---

## Key Takeaways

- Windows records important events automatically.
- Event Viewer is used to view these logs.
- Security logs are especially valuable for SOC Analysts.
- Event IDs help identify specific activities.

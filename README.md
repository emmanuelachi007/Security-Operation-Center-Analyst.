# Introduction to Logs

## Objective

Learn what logs are and why they are important in Cybersecurity and Security Operations Centers (SOC).

---

## What is a Log?

A log is a record of an event that occurs on a computer system, application, or network.

SOC Analysts use logs to investigate suspicious activities and identify security incidents.

---

## Example Log

Time: 14:30

User: john

Action: Login

Status: Failed

IP Address: 10.0.0.25

---

## Information Found in a Log

- Time
- Username
- Action Performed
- Status (Success or Failed)
- Source IP Address

---

## Why Logs are Important

Logs help security analysts:

- Detect brute-force attacks
- Investigate suspicious logins
- Monitor user activities
- Identify malware infections
- Perform incident investigations

---

## Example Security Scenario

If a user fails to log in over 100 times within five minutes from the same IP address, it may indicate a brute-force attack.

---

## Key Takeaways

- Logs record events.
- Every event has useful information.
- SOC Analysts investigate logs every day.
- SIEM tools collect and analyze logs automatically.

---

## Skills Learned

- Understanding Logs
- Reading Login Events
- Identifying Failed Logins
- Recognizing Brute-force Attacks

Windows Event Logs

## Objective

Learn the basics of Windows Event Logs and their importance in Security Operations Centers (SOC).

---

## What are Windows Event Logs?

Windows Event Logs record activities that occur on a Windows operating system.

These logs help administrators and SOC Analysts monitor systems, troubleshoot problems, and investigate security incidents.

---

## The Five Main Windows Logs

### 1. Application

Records events generated by applications.

Examples:
- Application crashes
- Installation errors
- Software warnings

### 2. Security

Records security-related events.

Examples:
- Successful logins
- Failed logins
- Password changes
- User account creation

### 3. System

Records events generated by Windows.

Examples:
- Driver issues
- Hardware failures
- Windows startup
- Windows shutdown

### 4. Setup

Records Windows installation and update events.

### 5. Forwarded Events

Stores logs collected from remote computers.

---

## Important Event IDs

4624 - Successful Login

4625 - Failed Login

4634 - User Logged Off

4720 - User Account Created

4726 - User Account Deleted

---

## Why SOC Analysts Use Event Logs

- Detect brute-force attacks
- Investigate login attempts
- Monitor system health
- Track user activity
- Investigate security incidents

---

## Key Takeaways

- Windows records important events automatically.
- Event Viewer is used to view these logs.
- Security logs are especially valuable for SOC Analysts.
- Event IDs help identify specific activities.

## Example Investigation

Scenario:

A user account generated multiple Event ID **4625** entries within a few minutes.

Possible Cause:

Brute-force attack.

1. Review the login logs.

#Splunk Fundamentals

## Objective

Understand the basics of Splunk and its role in a Security Operations Center (SOC).

---

## What is Splunk?

Splunk is a Security Information and Event Management (SIEM) platform that collects, stores, searches, and analyzes logs from computers, servers, firewalls, and applications.

---

## Why Organizations Use Splunk

- Collect logs from multiple systems.
- Detect suspicious activities.
- Investigate security incidents.
- Monitor system health.
- Generate alerts.

---

## Important Terms

### Data

Information collected from devices.

Examples:

- Windows Logs
- Linux Logs
- Firewall Logs

---

### Index

A storage location where Splunk saves logs.

---

### Event

A single log record.

Example:

User: Emmanuel

Action: Login

Status: Success

---

### Search

A query used to find information.

Example:

index=windows

---

## Basic SPL Searches

Search Windows Logs

index=windows

Search Failed Logins

index=windows EventCode=4625

Search Successful Logins

index=windows EventCode=4624

---

## Why Splunk is Important

Splunk helps SOC Analysts investigate incidents quickly by searching millions of log events in seconds.

---

## Key Takeaways

- Splunk is a SIEM tool.
- Splunk collects and searches logs.
- SPL is Splunk's search language.
- SOC Analysts use Splunk every day.

# Mission 19 - Splunk Stats Command

## Objective

Learn how to summarize and analyze log data using the Splunk `stats` command.

---

## What is the `stats` Command?

The `stats` command summarizes search results and helps SOC Analysts identify patterns quickly.

---

## Common Examples

### Count All Events

```spl
index=windows | stats count
```

### Count Failed Login Events

```spl
index=windows EventCode=4625 | stats count
```

### Count Events by User

```spl
index=windows | stats count by User
```

### Count Events by Source IP

```spl
index=windows | stats count by src_ip
```

---

## Why SOC Analysts Use `stats`

- Count security events.
- Identify the most active users.
- Detect suspicious IP addresses.
- Find unusual activity quickly.

---

## Key Takeaways

- `stats` summarizes data.
- `count` counts matching events.
- `by` groups results by a field.
- `stats` helps analysts investigate incidents efficiently.

7. Determine whether the activity is malicious.

- `count` counts matching events.
